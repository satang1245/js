원시타입 (Primitive Type), 객체타입 (Object Type or Reference Type)

원시타입

원시 타입의 값은 변경 불가능한 값이며, 값에 의한 전달이다.

# number

C언어 또는 Java등 일반적인 프로그램언어에서는 정수, 실수를 구분하여 Int, long, float, double 등과 같은 숫자 타입이 존재한다.
하지만 자바스크립트는 독특하게 하나의 숫자타입(Number)만 존재한다. (Chrome 67버전 이후 `BigInt` 타입이 추가되었다.)

*BigInt 알아보기

모든 숫자는 표기법만 다를 뿐 모두 정수가 아닌 실수로 표기한다.

var integer = 10;        // 정수
var double = 10.12;      // 실수
var negative = -20;      // 음의 정수
var binary = 0b01000001; // 2진수
var octal = 0o101;       // 8진수
var hex = 0x41;          // 16진수
console.log(binary); // 65
console.log(octal);  // 65
console.log(hex);    // 65

// 표기법만 다를뿐 같은 값이다.
console.log(binary === octal); // true
console.log(octal === hex);    // true

// 정수로 표시되지만 실제 처리는 실수로 처리한다.
console.log(1 === 1.0); // true

var result = 4 / 2;
console.log(result); // 2

Number 타입의 예외의 값들.

  • Infinity: 양의 무한대
  • -Infinity: 음의 무한대
  • NaN: 산술 연산 불가(not-a-number)
var pInf = 10 / 0;  // 양의 무한대
console.log(pInf);  // Infinity

var nInf = 10 / -0; // 음의 무한대
console.log(nInf);  // -Infinity

var nan = 1 * 'string'; // 산술 연산 불가
console.log(nan);       // NaN

# string

문자열(String) 타입은 텍스트 데이터를 나타내는데 사용하며,
작은 따옴표(') 또는 큰 따옴표(") 안에 텍스트를 넣어 생성한다.
*가장 일반적인 표기법은 작은 따옴표를 사용하는 것이다. 

var str = 'Hello';

// 문자열은 유사배열이다.
for (var i = 0; i < str.length; i++) {
    console.log(str[i]);
}

// 문자열을 변경할 수 없다.
str[0] = 'S';
console.log(str); // string

// 변수 재할당 가능
str += ' World';

# boolean

논리 타입으로 참(true) 또는 거짓(false)만 표현 가능하다.
*해당 타입은 조건(if)문에서 자주 사용된다.
*조건문에서는 null, undefined, 숫자(0)은 모두 false로 간주한다.
해당 관련글은 암묵적 타입변환에서 알아보자.

암묵적 타입변환에 대해 알아보기 
var foo = true;
var bar = false;

// typeof 연산자는 타입을 나타내는 문자열을 반환한다.
console.log(typeof foo); // boolean
console.log(typeof bar); // boolean

# undefined

undefined 타입의 경우 undefined의 유일한 값을 가진다.
변수 선언 이후 별도의 값을 할당하지 않을 경우나 존재하지 않는 객체 프로퍼티에 접근할 경우 해당 값을 반환한다.

var foo;
console.log(foo); // undefined

# null

null 타입의 경우 null의 유일한 값을 가진다.
변수의 값이 없다는 것을 명시적으로 표현할 때 사용하며,
함수나 DOM 검색등에서 유효한 값을 반환할 수 없을 경우 null을 반환한다.

*자바스크립트의 경우 대소문자를 구분하기 때문에, Null이나 NULL등과 다름으로 사용에 유의해야 한다. 
var element = document.querySelector('.myElem');
// HTML 문서에 myElem 클래스를 갖는 요소가 없다면 null을 반환한다.
console.log(element); // null

typeof를 이용하여 null 값을 연산해 보면 null이 아닌 object를 반환하는데, 이는 자바스크립트 자체적인 오류 이다.

// 연산시 `==`와 `===`의 차이를 주의하자.
typeof undefined     // "undefined"
typeof null; // "object"
null === undefined   // false
null  == undefined   // true
var foo = null;
typeof foo === null // false
foo === null // true

# symbol

ES6에서 새롭게 추가된 타입으로, 고유한 프로퍼티 키(property key)를 만들기 위해 사용된다.

var key = Symbol('key'); // Symbol(key)
var obj = { 'key' : 'value' }; // obj = { key: 'value' }

obj['key'] == 'value2'; // obj = { key: 'value2' }
obj[key] == 'value3'; // obj = { key: 'value2', Symbol(key): 'value3' }

객체타입

자바스크립트는 객체 지향 프로그래밍으로 설계된 언어로,
원시 타입을 제외한 모든 값(배열, 함수, 정규표현식)은 객체라 한다.
*객체는 키(key)와 값(value)로 이루어진 프로퍼티 들의 모음.

네이밍

좋은 코드란?
*주석을 단 설명이 없어도 변수 이름을 통하여 존재 목적을 쉽게 이해할 수 있는 소스.

// 😨 Bed
var x = 'satang1245'; // x는 아이디를 의미한다.
var y = '1987.12.25'; // y는 생년월일을 의미한다.

// 😄 Good
var userId = 'satang1245';
var birthday = '1998.12.25';

식별자 네이밍 규칙

  • 특수문자를 제외한 문자, 숫자, 언더스코어(_), 달러 기호($)를 포함할 수 있다.
    단, 예약어를 사용할 수 없으며, 숫자로 시작할 수 없다.
  • ES5부터 식별자를 만들 때, 유니코드 문자를 허용함으로 한글이나 일본어 식별자도 사용이 가능해졌지만 권장하진 않는다.
  • 일관성을 유지한다면 어떠한 네이밍 컨벤션을 사용해도 좋지만, 가독성을 높이기 위하여 ECMAScript사양에 정의되어 있는 네이밍 컨벤션을 따르는 것이 유리하다.
  • 일반적으로 ,

var FirstName;

// 헝가리언 케이스 (typeHungarianCase) - 이 표기법 개발자가 헝가리 사람이라 이렇게 지어졌다고 한다.
