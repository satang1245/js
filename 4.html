객체지향 프로그램밍이란
(OOP: Object-oriented Programming)

필요한 데이터를 추상화시켜 상태와 행위를 가진 객체를 만들고 객체들간의 유기적인 상호작용을 통해 로직을 구성하는 프로그래밍 방법.
우리가 주변의 사물을 인지하는 방식을 프로그래밍에 접목하려는 사상.
함수들의 집합 혹은 단순한 컴퓨터 명령어들의 목록이라는 절차지향 프로그래밍과 다른,
관계성 있는 객체들의 집합의 관점으로 접근하는 소프트웨어 디자인.
보다 유연하고 유지보수하기 쉬우며 확장성 측면에서 유리한 프로그래밍을 하도록 의도되었고,
주로 대규모 소프트웨어 개발에 널리 사용.

특징
  • 캡슐화
  • 추상화
  • 상속화
  • 다형성
장점
  • 코드 재사용성 용이
  • 유지보수의 우수성
  • 모듈에 분업 작업에 따른 업무 분담의 용이성
단점
  • 작업 난이도 상승(객체 처리에 대한 정확한 이해 필요)
  • 프로그램의 용량의 증가
  • 많은 시간 소요(클레스 및 객체 설계, 상속 등 구조적인 설계단계부터 많은 시간 소요)

객체란?

  • 원시타입을 제외한 모든 값(함수, 배열, 정규표현식 등)
  • 키(Key)와 값(Value)으로 구성된 프로퍼티(Property)들의 집합. 키는 다른 말로 속성이라고도 애기함.
  • 자바스크립트의 객체는 객체지향 상속을 구현하기 위해 "프로토타입(prototype)"이라고 불리는 객체의 프로퍼티와 메소드를 상속받을 수 있다.
    이 프로토타입은 타 언어와 구별되는 중요한 개념이다.
1.1 프로퍼티
  • 키 : 빈 문자열을 포함한 모든 문자열 또는 symbol 값
  • 값 : 함수를 포함한 모든 값

*프로퍼티 키에 문자열이나 symbol 값 이외의 값을 지정하면 암묵적으로 타입이 변환되어 문자열이 된다.
이미 존재하는 프로퍼티 키를 중복 선언하면 나중에 선언한 프로퍼티가 먼저 선언한 프로퍼티를 덮어쓴다.
배열과는 달리 객체는 프로퍼티를 열거할 때 순서를 보장하지 않는다.


1.2 메소드

프로퍼티 값이 함수일 경우, 일반 함수와 구분짓기 위해 메소드라 부른다.

객체 생성방법

객체생성 방법 
// 1. 객체 리터럴
var person1 = {};
person1.name = 'Lee';
person1.gender = 'male';

var person2 = {
  name: 'Lee',
  gender: 'male',
  sayHello: function () {
    console.log('Hi! My name is ' + this.name);
  }
};

//new 연산자
var person1 = new Object(); 
person1.sayHello = function () {
    console.log('Hi! My name is ' + this.name);
};


// 생성자 함수
function Person(name, gender) {
    this.name = name;
    this.gender = gender;
    this.hello = function() {
        console.log('Hi! My name is ' + this.name);
    };
}

// 인스턴스(instance) 생성
var person1 = new Person('Lee', 'male');
var person2 = new Person('Kim', 'female');
*인스턴스란?

예시) 붕어빵제작 틀 = 클래스, 붕어빵 재료들 = 객체, 붕어빵 완성품 = 인스턴스
소스예시) 
const myCar1 = new Car({
    name:'제네시스 GV70', 
    engine: '3.5', 
    color: 'blue'

const myCar2 = new Car({

객체에서 알아둬야 할 특징

참조타입 (Pass-by-reference)
객체의 모든 연산이 실제값이 아닌 참조값으로 처리됨을 의미한다. 원시 타입은 값이 한번 정해지면 변경할 수 없지만, 객체는 프로퍼티를 변경, 추가, 삭제가 가능하므로 변경가능 한 값이라 할 수 있다.

예시보기 
// pass-by-reference
var foo = {
    text: '안녕'
}
var bar = foo;
console.log( bar === foo, bar , foo );

bar.text = 'Hello';
console.log( bar , foo );

// 별개의 객체를 생성.
var foo = { text: '안녕' }
var bar = { text: '안녕' }
console.log( bar === foo, bar , foo );

bar.text = 'Hello';
console.log( bar , foo );

var foo = bar = { text: '안녕' }
console.log( bar === foo, bar , foo );

// pass-by-value - 원시타입의 경우 값은 복사 전달된다.
var foo = '안녕';
var bar = foo;

console.log( foo === bar, foo, bar );
bar = 'Hello';
console.log( foo === bar, foo, bar );

*변경 가능한 값(mutable value)과 변경 불가능한 값(immutable value)의 차이점 자세히 보기
