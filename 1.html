변수(Variable)

변수는 하나의 값을 저장하기 위해 확보한 메모리 공간 자체
또는 그 메모리 공간을 식별하기 위해 붙인 이름.

변수는 왜 사용하는가?

var userId = 1; // 할당 (변수명 = 변수값)
var userName = 'Lee';

변수는 연산된 결과값을 재사용 할 수 있도록 저장된 메모리 공간에 상징적인 이름을 붙인 것이다.
*변수명은 식별자라고도 불린다.

변수는 어디에 저장되는가?

변수는 자바스크립트 엔진내 관리영역인실행 컨텍스트에 저장되며,
실행 컨텍스트 내에 키(key) / 값(value) 형식인 객체로 등록되어 관리된다.

값

프로그램에 의해 조작될 수 있는 대상.

var str = 'Hello world';

위 예제에서 str = 변수, Hello World는 문자열 리터럴의 값 을 의미한다.

용어 의미
데이터 타입(Data Type) 프로그래밍 언어에서 사용할 수 있는 값의 종류
(Number, String, Boolean, null, undefined, Symbol)
변수(variable) 값이 저장된 메모리 공간의 주소를 가리키는 식별자(indentifier)
리터럴(literal) 소스코드 안에서 직접 만들어 낸 상수 값 자체를 말하며,
값을 구성하는 최소 단위
데이터타입 종류
원시타입 primitive data type
  • number
  • string
  • boolean
  • null
  • undefined
  • symbol (New in ECMAScript 6)
객체타입 object data type
  • Object
    • Array
    • Function
    • Regex(Regular Expression)
리터럴 종류 
// 숫자 리터럴
10.50
1001

// 문자열 리터럴
'Hello'
"World"

// 불리언 리터럴
true
false

// null 리터럴
null

// undefined 리터럴
undefined

// 객체 리터럴
{ name: 'Lee', gender: 'male' }

// 배열 리터럴
[ 1, 2, 3 ]

// 정규표현식 리터럴
/ab+c/

// 함수 리터럴
function() {}

문

문이란, 컴퓨터에 의해 프로그램인 스크립트가 단계별로 실행되는 명령들의 집합을 의미하며,
리터럴, 연산자, 표현식, 키워드 등으로 구성되고 세미콜론(;)으로 끝나야 한다.
*문의 끝에 붙이는 세미콜론은 자바스크립트 엔진이 스크립트를 해석할 때 문의 끝이라고 예측되는 지점에 세미콜론을 자동으로 붙여주는 세미콜론 자동 삽입 기능(ASI, automatic semicolon insertion)이 포함되어 있다. 하지만 세미콜론 자동 삽입 기능의 예측과 개발자의 예측과 다른 경우가 간혹 있어서 사용에 주의해야 한다.

문은 블럭으로 그룹화 할 수 있다. (추후 스코프와 관련있기 때문에 알아둬야 함)

// 함수
function myFunc(x,y) {
    return x + y;
}

// if 문
if (x > 50) {
    // do something..
}

// for 문
for (var i=0; i<10; i++) {
    // do something..
}

    a = 'a';
} else if ( 1 < -1) {
    a = 'b';

    a = 'a';
} else if ( 1 < -1) {
    a = 'b';
}
console.log(a); // ??

상수(constant) ??? 😳

변수 - 값의 재할당이 가능한 것을 말하며, 상수는 값의 재할당이 불가능한 것을 말한다.
*단, const 키워드로 선언된 변수가 객체인 경우, 값을 변경 할 수 있다.
const 키워드는 재할당을 금지할 뿐 "불변"을 의미하지 않는다.
