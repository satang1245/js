Single Page Application

SPA

최조 접근시 아무행동이 일어나지 않고, 동적으로 서버에 데이터를 요청하여
화면에 가상의 DOM을 생성하여 화면을 랜더링 함.

[장점]
  • 고성능의 애플리케이션을 제공
  • 네이티브 애플리케이션에 대해서 API를 통한 느슨한 결합의 시스템을 구성해 대응이 가능
  • JSON API를 통해 느슨한 결합의 설계 가능
[단점]
  • 초기 렌더링이 발생하며, 이에 대한 대책이 없으면 다소 시간이 오래 걸릴 수 있다
  • 학습 비용이 높다
  • 애플리케이션 제작시 프론트엔드의 비중이 높아진다
[특징]
  • URL 경로와 뷰의 라우팅 관리
  • 브라우저 이력 관리를 통한 페이지 이동
  • 비동기를 통한 데이터 얻기
  • 뷰 렌더링
  • 모듈화된 코드 관리
*React는 View에 특화된 라이브러리 이며, React Router와 결합하여 SPA를 구성한다.
